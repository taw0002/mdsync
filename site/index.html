Open source CLI tool

Every AI agent outputs markdown.
Reading it sucks. We fixed that.

One command. Your markdown opens in the browser with real typography, code highlighting, and inline editing that saves back to the file.

Try the demo View on GitHub
Get started
npx markdownsync README.md
What happens
Markdown in, styled document out.

Point it at a file or a folder. It starts a local server and opens your browser. Edit in the browser or in your text editor — changes sync both ways in real time.

mdsync README.md mdsync ./docs mdsync build ./docs

Render anything

Works on any .md file or folder. Headings, code blocks, tables, task lists, links. Dark and light themes.

Edit in place

Double-click any section. Edit with a rich text toolbar. Hit save. The original .md file updates on disk.

Two-way sync

Change the file in VS Code and the browser updates. Change it in the browser and the file updates. No refresh needed.

MCP server

Run mdsync mcp to expose your docs to Claude, Cursor, or any MCP-compatible AI tool.

What people use it for

Docs, notes, and everything AI agents generate.

Project docs

mdsync ./docs — browse, search, and edit your project's documentation without leaving the terminal workflow.

Personal notes

mdsync ~/notes — a lightweight note viewer for plain markdown files. No vault format, no proprietary sync.

AI output review

AI agents dump walls of markdown. mdsync makes it readable and lets you edit the parts that need fixing.

Static docs site

mdsync build ./docs --out ./site — generate a static site from the same markdown. Deploy anywhere.
